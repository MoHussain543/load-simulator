How it works

How the test runs

  1. You choose a local URL, virtual users, and duration.
  2. Each virtual user sends repeated GET requests until the timer expires.
  3. The backend waits for the full run, then returns aggregated metrics.

Success vs failure

HTTP 2xx responses count as successful. 4xx, 5xx, connection errors, and timeouts count as failed.

Metric glossary

Total Requests
All completed requests (successful + failed).
Successful
Requests that returned HTTP 2xx.
Failed
Requests that returned 4xx/5xx or could not complete.
Avg / Min / Max Response
Mean, fastest, and slowest response times in milliseconds.
P95 Response
95th percentile latency — 95% of requests were faster than this.
Requests / sec
Throughput: total requests divided by the configured duration.
Error Rate
Percentage of failed requests out of the total.

Only http://localhost and http://127.0.0.1 targets are allowed.
